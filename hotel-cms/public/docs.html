Build and manage hotel websites across your entire portfolio with AI that understands your brand. Multi-property management, intelligent content generation, campaign syndication, competitive intelligence, guest review AI, and one-click translations.

hotelCMS is a full-stack platform that combines a visual page builder with a multi-model AI engine designed for hotel chains. Hotel teams manage their websites through a CMS dashboard, while AI interprets natural-language commands to modify content, swap components, adjust themes, translate pages, and inject custom code — all version-controlled with instant rollback.

v3.0 adds four major capabilities: AI Content Syndication for cross-property campaigns, Competitive Intelligence for monitoring rival hotels, Guest Review Response AI for sentiment-adjusted review replies, and Brand Governance for org-wide theme and section locking. Manage one property or an entire fleet from a single dashboard.

Tech Foundation

Built on Next.js 16 with App Router, Server Components, and Turbopack. PostgreSQL with Prisma ORM for typed database access. OpenRouter as the LLM gateway using nvidia/nemotron-3-super-120b-a12b:free. Tailwind CSS for component styling. bcryptjs for password hashing.

Tier Routing Logic

Tier Trigger Handler Latency
Tier 0 Rule-matched: simple text/prop updates, meta changes Direct DB write, no LLM call < 50ms
Tier 1 Structured CMS ops: swap, add/remove section, reorder Fast, cheap LLM ~400ms
Tier 2 Multi-step ops: theme changes, memory, brand voice, translations Mid-tier LLM ~1.5s
Tier 3 Code injection, HTML replacement, URL reference, complex reasoning Frontier LLM (Nemotron 120B) ~4s

Getting Started

Get hotelCMS running locally in under 5 minutes.

1
Install PostgreSQL
hotelCMS requires PostgreSQL 14+. Install via Homebrew (macOS), apt (Ubuntu), or use a managed service like Neon or Supabase.
# macOS
brew install postgresql@16 && brew services start postgresql@16

# Ubuntu
sudo apt install -y postgresql postgresql-contrib
2
Clone the repository and install dependencies
git clone https://github.com/your-org/hotel-cms.git
cd hotel-cms
npm install
3
Configure environment variables
Create a .env file in the project root with these required variables.
# .env
DATABASE_URL="postgresql://user:password@localhost:5432/hotelcms"
OPENROUTER_API_KEY="sk-or-..."
4
Run database migrations
npx prisma migrate dev
5
Seed demo data
Creates a "Demo Hotel Group" organization, sample hotel, pages, components, and an admin user.
npx prisma db seed
6
Start the development server
npm run dev
7
Open hotelCMS
Navigate to http://localhost:3000 — it redirects to /login.
http://localhost:3000
8
Log in with the seeded admin account
Email:    admin@hotelcms.com
Password: admin123

User Roles

editor Content editing: pages, sections, rooms, media, AI commands, review responses Only assigned hotels via hotelAccess JSON array
viewer Read-only: view dashboard, preview pages, view competitor insights Only assigned hotels via hotelAccess JSON array

Session-Based Auth

Authentication uses httpOnly cookies with session tokens stored in the database. No JWT — sessions can be revoked server-side instantly.

  • POST /api/auth/register — Create account (assigns to organization)
  • POST /api/auth/login — Validate credentials, create session
  • POST /api/auth/logout — Destroy session
  • GET /api/auth/me — Return current user from session

Team Management

Admins can invite users, assign roles, and control hotel-level access from the Users tab. Each user's hotelAccess field is a JSON array of hotel IDs they can access.

CMS Operations Reference

The AI emits one or more of these 18 typed operations in response to every user command. Operations are validated with Zod and executed atomically.

Content Operations

Operation Description Min Tier Key Payload Fields
update_text Change visible text content in a section (headline, body, CTA label) Tier 0 sectionId, field, value
update_props Update structured props (layout variant, colors, image URLs, boolean flags) Tier 0 sectionId, props (partial)
swap_component Replace a section's component type while preserving compatible props Tier 1 sectionId, newComponentId
add_section Insert a new section at a specified position on the page Tier 1 pageId, componentId, position, props
remove_section Delete a section from a page (soft delete, recoverable via rollback) Tier 1 sectionId
update_theme Modify global theme tokens: colors, typography, spacing, radius Tier 2 hotelId, themeChanges (partial Theme)
update_meta Update page SEO metadata: title, description, OG image, canonical Tier 0 pageId, meta
reorder_sections Resequence sections on a page by providing a new ordered array of IDs Tier 1 pageId, sectionIds (ordered)

Code Editing Operations

Operation Description Min Tier Key Payload Fields
inject_css Append scoped custom CSS targeting a specific section or global scope Tier 3 sectionId?, css
replace_html Fully replace a section's rendered HTML with AI-generated markup Tier 3 sectionId, html
clear_overrides Remove all custom CSS/HTML overrides for a section, revert to component defaults Tier 1 sectionId

Context & Memory Operations

Operation Description Min Tier Key Payload Fields
save_memory Persist a fact or preference to the hotel's AI memory store for future sessions Tier 2 hotelId, key, value
update_brand_voice Update the hotel's brand voice descriptor used in all subsequent AI content generation Tier 2 hotelId, brandVoice

Translation Operations

Operation Description Min Tier Key Payload Fields
translate_site Clone all pages for a locale, AI translates text content across every section Tier 2 hotelId, targetLocale
ℹ
A single AI response can contain multiple operations that execute in order. For example, swapping a component and then updating its props happens in one atomic transaction. The LLM approach selection rules determine which tier handles the full batch.

Smart Links

A centralized link registry per hotel. Tokens auto-resolve everywhere across all components, so you never hard-code URLs.

Available Tokens

{{booking}} Booking engine URL
{{phone}} Primary phone number
{{email}} Contact email address
{{maps}} Google Maps link
{{whatsapp}} WhatsApp chat link
{{instagram}} Instagram profile
{{facebook}} Facebook page
{{twitter}} Twitter/X profile
{{page:slug}} Internal page by slug

How It Works

Tokens are auto-derived from the hotel's contactInfo JSON field. When a component renders {{booking}}, the renderer resolves it to the hotel's actual booking URL. Update the link in one place and it propagates everywhere.

The LinksEditor in the CMS sidebar allows editing all links for a hotel in one panel. Changes take effect immediately in the preview iframe.

Translation Flow

1 — Trigger translation
User says "Translate to French" or clicks Add Language in the CMS locale switcher. Calls POST /api/i18n/translate/[hotelId].
2 — Clone pages
All pages are cloned with a locale suffix. Section props are deep-copied. Structure is identical to the original.
3 — AI batch translate
All translatable text fields across all sections are sent to the LLM in a single batch. AI preserves HTML structure and only translates visible text.
4 — Store per-locale
Translated pages are stored with their locale code. Preview at /preview/[hotelId]/lang/[locale]/[pageSlug].

Preview & Switching

The preview renderer includes a language switcher widget that lets you toggle between locales in real-time. The CMS also has a locale switcher and an Add Language modal for triggering new translations.

To remove a locale: DELETE /api/i18n/[hotelId]/[locale]

AI Content Syndication v3.0

Write one campaign brief, AI adapts the content for each target property. Full lifecycle: create, launch, monitor, and end.

Campaign Lifecycle

1 — Create campaign
Set a title (e.g. "Summer Escape 2026"), write a brief describing the promotion, and select target hotels. Stored as a Campaign record with status draft.
2 — Launch campaign
AI reads the brief and each hotel's context, then generates tailored promotional content per property. A boutique beach resort gets different messaging than a downtown business hotel. Status moves to active.
3 — Monitor deployment
The campaign dashboard shows per-hotel deployment status: pending, deployed, or failed. Retry available for failures.
4 — End campaign
Click "End Campaign" to remove promotional content from all participating hotels in one step. Status moves to ended. Clean removal, no manual cleanup.

Data Model

The Campaign Prisma model stores: title, brief, hotelIds (JSON array), status (draft/active/ended), deployments (JSON with per-hotel results), and organizationId.

API Endpoints

  • GET /api/campaigns — List all campaigns for the organization
  • POST /api/campaigns — Create a new campaign
  • GET /api/campaigns/[id] — Get campaign details with deployment status
  • PUT /api/campaigns/[id] — Update campaign title, brief, or target hotels
  • DELETE /api/campaigns/[id] — Delete a campaign
  • POST /api/campaigns/[id]/launch — Launch: AI generates and deploys content
  • POST /api/campaigns/[id]/end — End: removes campaign content from all hotels

Competitive Intelligence v3.0

Monitor competitor hotel websites. Detect changes and get AI-generated strategic counter-move suggestions.

How It Works

1 — Add competitors
Enter a competitor's name and website URL. The competitor is linked to a specific hotel in your portfolio for targeted analysis.
2 — Scan competitor sites
Trigger a scan manually or for all competitors at once. The system fetches the competitor's website, extracts content, and stores a snapshot as a CompetitorScan.
3 — AI analysis
AI compares the competitor's content against your hotel's site and generates insights: what they changed, what they do better, and recommended counter-moves.
4 — Track over time
Scan history is preserved. Compare snapshots over time to detect pricing changes, new promotions, or content strategy shifts.

Data Model

Competitor stores: name, websiteUrl, hotelId, notes, lastScannedAt. CompetitorScan stores: competitorId, content (extracted text), insights (AI-generated JSON), scannedAt.

API Endpoints

  • GET /api/competitors — List competitors (filtered by hotel)
  • POST /api/competitors — Add a new competitor
  • GET /api/competitors/[id] — Get competitor details with latest scan
  • PUT /api/competitors/[id] — Update competitor info
  • DELETE /api/competitors/[id] — Remove a competitor
  • POST /api/competitors/[id]/scan — Trigger a scan for one competitor
  • POST /api/competitors/scan-all — Scan all competitors for a hotel
  • GET /api/competitors/[id]/history — Get scan history with snapshots

Guest Review Response AI v3.0

Paste guest reviews from any platform. AI drafts on-brand responses with sentiment-adjusted tone. Analytics dashboard for tracking review health.

Response Generation Flow

1 — Add review
Paste the guest name, rating (1-5), platform (Google/TripAdvisor/Booking.com/etc.), and review text. AI immediately analyzes sentiment.
2 — AI drafts response
Using the hotel's brand voice and the review's sentiment, AI generates an appropriate response. Positive reviews get warm acknowledgment; negative reviews get empathetic, resolution-focused replies.
3 — Review and copy
Edit the response if needed, then copy to clipboard and paste directly into the review platform. Regenerate option available for alternative wording.

Sentiment Analysis

Each review is tagged with a sentiment: positive, neutral, or negative. The analytics dashboard shows aggregate sentiment distribution, average rating, and response rate for each hotel.

Data Model

Review stores: guestName, platform, rating, reviewText, sentiment, aiResponse, status (pending/responded), hotelId.

API Endpoints

  • GET /api/reviews — List reviews for a hotel (with filtering/sorting)
  • POST /api/reviews — Add a review and trigger AI response generation
  • GET /api/reviews/[id] — Get review details with AI response
  • PUT /api/reviews/[id] — Update review or response
  • DELETE /api/reviews/[id] — Delete a review
  • POST /api/reviews/[id]/regenerate — Regenerate AI response with alternative wording
  • GET /api/reviews/stats — Get sentiment analytics per hotel

Brand Governance Engine v3.0

Org-level brand themes and locked sections cascade to all hotels. AI protection ensures locked content stays consistent across the fleet.

How It Works

Brand governance operates at the organization level. Admins define a master brand theme (colors, typography, spacing) and designate locked sections (e.g., the footer, legal disclaimers, brand header) that cascade to every hotel in the organization.

Brand Theme Cascade

When an org-level brand theme is set, it overrides individual hotel themes for the specified tokens. Hotels can still customize non-locked tokens, but the brand-level values take precedence for consistency.

Locked Sections

Locked sections are defined at the org level and pushed to all hotels. Editors cannot modify locked sections, and AI commands that target locked sections are blocked with a clear error message. Only admins can update locked sections through the brand governance panel.

AI Protection

The AI engine checks for locked sections before executing any operation. If a user tries to modify a locked section via the command bar, AI returns an explanatory message instead of making the change. This prevents accidental brand drift across properties.

API Endpoints

  • GET /api/brand — Get org-level brand theme and locked sections
  • PUT /api/brand/theme — Update the org-level brand theme tokens
  • PUT /api/brand/sections — Update locked section definitions
